# Rendermind Brand Guidelines v1.1

## 1. Identity & Vision

**Name:** Rendermind

**Primary positioning:**  
Embedded media workflows for AI video, SaaS, and marketing teams.

**Slogan:**  
Reusable media infrastructure for custom creation workflows.

**Alternative slogan:**  
Embedded editing and rendering infrastructure for AI and SaaS products.

**Mission:**  
Help product teams launch custom image and video creation workflows faster through reusable editors, rendering pipelines, captioning modules, automation workflows, and AI-ready media infrastructure.

**Vision:**  
Become the technical foundation behind the next generation of creative, media, and AI-powered product experiences.

Rendermind should feel like a specialized engineering partner, not a generic SaaS template or a traditional agency.

---

## 2. Brand Strategy

### 2.1 What Rendermind is

Rendermind is a specialized technical service powered by reusable SDKs, rendering engines, and custom integration work.

It helps teams integrate:

- embeddable image and video editors;
- server-side image, PDF, and video rendering;
- template-based generation workflows;
- transcription and captioning pipelines;
- AI-ready media workflows such as semantic transcript search.

### 2.2 What Rendermind is not

Rendermind should not be positioned as:

- a fully self-serve plug-and-play SaaS platform;
- a generic no-code editor;
- a design tool for end users;
- a traditional software agency;
- an AI company using vague AI claims.

The correct positioning is:

> Reusable media modules + custom engineering + product-specific integration.

## 4. Messaging Principles

### 4.1 Core message

Rendermind helps product teams integrate custom image and video workflows without building the entire media stack from scratch.

### 4.2 Messaging pillars

**1. Reusable modules**  
Rendermind provides real editor, renderer, captioning, and AI-ready building blocks that can be adapted to specific workflows.

**2. Custom integration**  
Rendermind is built around the customer’s existing product, auth, UX, billing, data model, and business logic.

**3. Faster path to launch**  
Customers can start with the workflow blocking their roadmap instead of building a full media platform internally.

**4. Technical depth**  
The brand is grounded in real experience building graphic editors, video editors, template systems, rendering pipelines, and media automation flows.

**5. AI-ready, not AI-hyped**  
AI should be positioned as part of useful media workflows: captions, transcripts, semantic search, prompt-to-template flows, review, and rendering.

### 4.3 Preferred language

Use:

- reusable SDKs;
- custom integration;
- module-based implementation;
- production-tested modules;
- AI-ready media workflows;
- embedded editors;
- rendering pipelines;
- technical discovery;
- product-specific workflows;
- media infrastructure;
- launch the first workflow faster.

Avoid or use carefully:

- plug-and-play;
- fully managed platform;
- AI-native;
- enterprise-ready platform;
- everything you need;
- production-ready infrastructure;
- go live in weeks, not quarters;
- no-code;
- magic;
- autonomous AI.

## 5. Voice & Tone

The brand voice should be:

- technical;
- precise;
- premium;
- pragmatic;
- confident;
- calm;
- founder/CTO-friendly.

It should not feel:

- overhyped;
- gimmicky;
- overly futuristic;
- agency-generic;
- too academic;
- too playful.

### Writing style

Use short, direct sentences.

Prefer:

> Start with the workflow blocking your roadmap.

Avoid:

> Unlock the future of limitless AI-powered creative transformation.

Prefer:

> Reusable media modules, integrated around your product.

Avoid:

> The ultimate all-in-one creative infrastructure platform.

---

## 7. Color Palette

### 7.1 Core colors

**Deep Obsidian**  
Primary background.  
Use for the main site/app background.  
Suggested value: `oklch(0.145 0.012 222)`  
Legacy hex reference: `#0B0E14`

**Elevated Obsidian**  
Card and surface background.  
Use for cards, code blocks, panels, and elevated sections.  
Suggested value: `oklch(0.18 0.018 222)`

**Accent Slate**  
Secondary surfaces, separators, muted panels.  
Suggested value: `oklch(0.25 0.018 222)`  
Legacy hex reference: `#1E2228`

**Electric Cyan**  
Primary brand action color.  
Use for primary CTAs, focus rings, important highlights, and selected states.  
Suggested value: `oklch(0.84 0.20 195)`  
Legacy hex reference: `#00F5FF`

**Soft Cyan**  
Secondary accent color.  
Use for badges, glows, subtle accents, and technical labels.  
Suggested value: `oklch(0.78 0.16 195)`

**Ghost White**  
Primary typography.  
Suggested value: `oklch(0.96 0.008 222)`  
Legacy hex reference: `#F1F1F1`

**Tech Silver**  
Secondary typography and labels.  
Suggested value: `oklch(0.78 0.012 222)`  
Legacy hex reference: `#8E949E`

### 7.2 Usage rules

- Use cyan sparingly.
- Reserve the strongest cyan for primary CTAs and key highlights.
- Keep most surfaces dark, but ensure cards are visibly elevated from the background.
- Avoid pure black backgrounds.
- Avoid using cyan for large blocks of text.
- Use subtle gradients/glows to create depth, not decoration.

## 8. Typography

### 8.1 Font families

**Headings:** `Space Grotesk`  
Use for hero headlines, section titles, and strong product statements.

**Body:** `Inter`  
Use for paragraphs, cards, forms, navigation, and general UI.

**Code / Technical:** `JetBrains Mono`  
Use for code snippets, technical specs, labels, package names, and small technical annotations.

### 8.2 Type style

Headings should feel geometric, sharp, and confident.

Recommended heading style:

- tight tracking;
- strong contrast;
- short lines;
- clear hierarchy.

Body copy should feel readable, calm, and precise.

Avoid long paragraphs. Prefer short sections, cards, and scannable blocks.

---

## 9. UI System

### 9.1 Surfaces

Use layered dark surfaces:

- page background;
- elevated cards;
- code panels;
- CTA panels;
- subtle section backgrounds.

Cards should use:

- visible but subtle borders;
- soft shadows;
- slight surface contrast;
- optional hover elevation for interactive elements.
